## Definition

Agent planning is the process by which an agent breaks a high-level goal into a sequence of steps — identifying which tool each step needs, whether it is read-only or write, which steps require human confirmation, and how task completion will be verified. A plan generated by the model is a candidate route, not an authorization.

## Key Points

- A good agent plan must expose: which tool each step needs, whether each step reads or writes, which steps run automatically, which steps require user confirmation, whether failure can be retried, and how final task completion will be verified
- **The plan is a proposal, not a permission** — the system must check each step against policy before execution
- "The closer a plan gets to high-risk action, the more the plan must be broken apart and checked by system rules"
- The distinction between read-only and write steps is the primary planning-layer security boundary — read steps can run automatically; write steps must enter policy checks and potentially simulation
- In AI × Web3 agent architecture: the system splits the plan into read-only steps and candidate write steps before any execution begins (step 3 of the 8-step architecture)
