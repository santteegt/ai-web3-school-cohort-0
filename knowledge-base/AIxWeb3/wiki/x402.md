## Definition

x402 turns HTTP 402 Payment Required into an internet-native payment flow, allowing services to respond with payment requirements that agents handle automatically. The client requests a resource, the service returns a payment requirement, and the client re-requests with proof of payment after completing the payment.

## Key Points

- Suitable for small, per-use payments: APIs, data, content, agent services
- Puts "payment required to access resources" back into HTTP semantics — tools handle 402 like 401 auth
- For agents: read payment requirement → check budget → complete payment → replay request
- x402 only solves part of the problem: does NOT automatically solve service quality, refunds, delivery disputes, or long-term subscriptions
- High-value services still require [[escrow]], [[receipt]], and [[dispute]] mechanisms
