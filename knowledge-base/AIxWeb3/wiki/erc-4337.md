# ERC-4337 (Account Abstraction)

## Definition

ERC-4337 is the Ethereum standard for account abstraction — enabling "smart accounts" (contract wallets) that can have programmable authorization logic, sponsor gas, support social recovery, and use session keys. It achieves this without requiring protocol changes by introducing a separate UserOperation mempool and an EntryPoint contract.

## Key Points

- **UserOperation**: a new transaction-like object submitted by users to the alt mempool; contains calldata, gas params, and a signature for the smart account's validation logic
- **EntryPoint contract**: the singleton on-chain contract that validates and executes UserOperations; all ERC-4337 accounts interact with it
- **Bundler**: off-chain node that collects UserOperations from the alt mempool, bundles them, and submits them to the EntryPoint as a single on-chain transaction
- **Paymaster**: contract that can sponsor gas fees; enables gas-less UX or gas paid in ERC-20 tokens
- **Smart account validation**: the account contract defines its own `validateUserOp` function — can implement multisig, passkeys, session key limits, etc.
- **Session keys**: limited-scope keys granted to AI agents or dApps; time-bounded, contract-scoped, amount-capped
- **Safe** is a major smart account implementation; many teams use Safe + ERC-4337 modules
- **Key AI × Web3 infrastructure**: removes the need for agents to hold raw private keys; enables human-confirmable, permission-bounded agent actions
