## Summary

This source provides a framework for evaluating and choosing a direction within the AI × Web3 space. It defines six foundational directions, a five-criteria direction evaluation matrix, a step-by-step process for choosing one main direction, and deep exploration paths for each direction (Payment, Identity/Capability, Wallet, Privacy/Security, Governance). The central thesis is that valid AI × Web3 problems require both AI capabilities AND Web3 mechanisms to be indispensable — not merely co-present in a title.

## Notable Points

- "Truly valuable problems usually sit at the intersection of machine execution + economic exchange + permission control + verifiable records."
- Counterexample for payment: if an agent merely helps click a traditional payment button without budget control, proof of delivery, or dispute handling, it is closer to ordinary automation.
- Counterexample for wallet: if an AI wallet can only show "send transactions in natural language" without explaining permission limits and failure handling, it is a dangerous demo, not a reliable direction.
- Counterexample for governance: if a governance assistant auto-generates and passes budget proposals without human confirmation and accountability, it is a governance risk, not an efficiency improvement.
- Each direction requires choosing only ONE main focus — do not open too many threads at once.
