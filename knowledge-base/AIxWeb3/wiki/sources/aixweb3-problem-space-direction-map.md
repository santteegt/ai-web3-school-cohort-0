## Summary

This source provides a framework for evaluating and choosing a direction within the AI × Web3 space. It defines six foundational directions (payment/commerce, identity/reputation, capability/interoperability, wallet/permission, privacy/security, governance/coordination), a five-criteria direction evaluation matrix, a four-step process for selecting one main direction, and detailed deep exploration paths for each direction. The central thesis is that valid AI × Web3 problems require both AI capabilities AND Web3 mechanisms to be indispensable — not merely co-present in a title. Each direction comes with a structured decomposition template and an explicit counterexample showing what does NOT qualify as a strong AI × Web3 direction.

## Notable Points

- "Truly valuable problems usually sit at the intersection of machine execution + economic exchange + permission control + verifiable records."
- Counterexample for payment: if an agent merely helps click a traditional payment button without budget control, proof of delivery, or dispute handling, it is closer to ordinary automation.
- Counterexample for wallet: if an AI wallet can only show "send transactions in natural language" without explaining permission limits and failure handling, it is a dangerous demo, not a reliable direction.
- Counterexample for governance: if a governance assistant auto-generates and passes budget proposals without human confirmation and accountability, it is a governance risk, not an efficiency improvement.
- Counterexample for identity: if a system only issues an NFT business card to an agent without capability claims, delivery records, or verification methods, it does not form a complete identity/reputation direction.
- Each direction requires choosing only ONE main focus — do not open too many threads at once.
