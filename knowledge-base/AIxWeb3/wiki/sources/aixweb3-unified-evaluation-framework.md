## Summary

This source provides a 7-question framework for evaluating any AI × Web3 project direction. The questions are designed to be applied consistently across all directions — payment, identity, wallet, privacy, governance — to prevent projects from merely co-locating "AI" and "Web3" without genuine interdependence. The framework addresses: necessity of each technology, stakeholder roles, automation vs. human confirmation boundaries, verification cost, project layer, and failure mode analysis.

## Notable Points

- Question 1 + 2 together form the "mutual necessity test": would this problem still exist without AI? Without Web3? Both must answer "no" for the direction to be genuine AIxWeb3.
- Question 4 (which actions require human confirmation) directly maps to the human-in-the-loop requirement for wallet/permission directions.
- Question 7 (failure mode analysis) prevents direction selection based on optimism — forces naming the most likely failure: no demand, trust failure, cost too high, immature interfaces, permission risk, or workflow resistance.
