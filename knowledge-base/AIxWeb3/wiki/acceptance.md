## Definition

Acceptance is the action by the payer or rule system confirming that delivery meets requirements. High-value tasks should not rely only on a model saying "it looks finished" — clear acceptance criteria, deadlines, and failure paths are needed.

## Key Points

- Acceptance can be: user click, automatic evaluator, or a combination
- Break down into checkable conditions: fields complete, tests passed, submitted within time, hashes match, budget not exceeded
- Subjective judgments can exist but must indicate who made them
- Recommended for AI acceptance: "AI preliminary review + challenge window + human review" to handle disputes
- Low-risk tasks can pass automatically; high-dispute tasks should not be decided by a single model
