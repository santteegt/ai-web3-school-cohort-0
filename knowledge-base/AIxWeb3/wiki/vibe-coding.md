## Definition

Vibe coding is human-AI collaborative software iteration where humans own direction, constraints, and acceptance — and AI agents handle generation, modification, search, and routine engineering actions. Its core value is shortening the engineering feedback loop; its core risk is that speed without discipline amplifies disorder.

## Key Points

- **"Vibe coding is NOT coding by feeling"** — it requires more rigorous task scoping, context accuracy, test coverage, and commit discipline, not less
- "AI cannot take over engineering judgment for you" — task description clarity, result review, change scope control, and bug verification remain human responsibilities
- **Small tasks outperform big rewrites**: the smaller the task, the more reviewable the result; prefer local patches over untraceable large changes
- "Put AI Coding into normal engineering flow: issue → branch → commit → test → review → merge → release — do NOT let AI bypass these flows"
- Good agent participation in engineering: extract task boundaries from issues, search related files, generate smallest patch, run tests and explain failures, write PR summary
- In AI × Web3 School, vibe coding with Hermes Agent is a required Week 1 hands-on exercise
- In AI × Web3: useful for hackathon exploration and prototyping, but smart contract code requires review, simulation, and multi-party confirmation regardless of how it was generated
