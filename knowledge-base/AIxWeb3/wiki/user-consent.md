## Definition

User Consent ensures users clearly know how their data will be used — with specific, revocable, per-action toggles rather than blanket agreements. Connecting a wallet is not consenting to full history analysis; using an agent is not consenting to all third-party model data sharing.

## Key Points

- Granular consent examples: "allow reading current address balance," "allow analyzing last 30 days of transactions," "allow saving risk preferences," "allow sending summaries to cloud models" — separate toggles
- Consent must be revocable: explain what data gets deleted, what public on-chain records cannot be deleted, what logs must be kept for security/compliance
- Blanket "agree to all" is not meaningful consent in an AI × Web3 context
- Connects to data portability: users should be able to export data they consented to share
