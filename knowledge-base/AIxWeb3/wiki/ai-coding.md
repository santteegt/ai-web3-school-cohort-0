## Definition

AI coding is the practice of using AI agents (Claude Code, Cursor, Codex CLI) to accelerate the software engineering loop — agents handle generation, modification, search, and routine engineering actions while humans own direction, constraints, and acceptance criteria.

## Key Points

- "The core of AI Coding is shortening the engineering feedback loop" — speed is the goal, but speed without discipline amplifies disorder
- The real questions once speed increases: can you describe tasks clearly? can you review results? can you control change scope? can you verify no new bugs were introduced?
- **AI Coding is not "coding by feeling"** — it requires more rigorous repo management, task scoping, context accuracy, tests, and commit boundaries, not less
- Good AI coding agent participation in engineering: extract task boundaries from issues, search related files, generate smallest patch, run tests and explain failures, supplement changelog/docs, write PR summary
- **"Put AI Coding into normal engineering flow: issue → branch → commit → test → review → merge → release. Do NOT let AI bypass these flows"**
- Configuration should be team-shared and reusable, not individual machine tweaks
- In AI × Web3: smart contract code generated by AI must go through review, simulation, and multi-party confirmation — chain-related code is higher risk
