## Definition

Verifiable AI is the set of techniques and system designs that allow model outputs, agent execution processes, and results to be independently verified — ensuring that claims about what an AI did or decided can be proven, not just trusted.

## Key Points

- Verifiability can apply at three levels: output (was this text generated by this model?), process (did the agent follow this workflow?), result (did the on-chain action match the stated intent?)
- In AI × Web3, on-chain records (transaction hashes, event logs, receipts) provide the strongest verifiability for agent actions
- Cryptographic proofs (ZK proofs, attestations) can extend verifiability to model execution itself — proving a specific model was used with specific inputs
- Verifiable AI connects to [[agent-identity]]: knowing WHO did something AND being able to prove HOW they did it
- Without verifiable AI, AI-assisted governance, AI-managed funds, or AI-executed contracts require complete blind trust in the AI operator
