## Definition

The Privacy, Security, and Sovereignty direction covers the full attack surface and control landscape once an agent holds context, credentials, API keys, private keys, or a budget. Security is a system prerequisite, not a side issue.

## Key Points

- Five risk categories: prompt injection, tool abuse, unauthorized execution, sensitive data leakage, and dependency on model providers — each can expand into additional attack surfaces
- Security design must answer: what can the agent see, what can it call, how much can it spend, on whose behalf can it make decisions, and who is responsible when something goes wrong?
- Privacy/sovereignty is not just "local model" — it involves data boundaries, provider dependency, portability, auditing, and user control
- Systems requiring users to entrust private keys, full transaction permissions, and complete context to a black-box agent must be marked high-risk
