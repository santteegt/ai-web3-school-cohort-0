## Definition

Data Portability allows users to take their preferences, memories, task history, and credentials with them when they switch platforms. Memory that cannot be exported is a form of lock-in — the long-term value of an agent tied to non-portable context is a platform advantage, not a user benefit.

## Key Points

- Layered export is more practical than dumping all raw data: task history, preferences, tool logs, identity credentials, public reputation, private memories — each with sensitivity level marked
- For use by another agent: machine-readable formats (JSON schemas, VCs, signed logs, profile files)
- "Exportable" that only means downloading text humans can't parse is not meaningful portability
- Connects to censorship resistance: if identity and context are portable, users can migrate away from any platform
