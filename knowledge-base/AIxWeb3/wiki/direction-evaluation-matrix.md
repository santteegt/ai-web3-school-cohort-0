## Definition

The Direction Evaluation Matrix is a five-criteria framework for assessing whether an AI × Web3 direction is worth pursuing. It prevents pursuing directions that sound technically interesting but lack structural demand, verifiability, or a tractable entry point.

## Key Points (Five Criteria)

1. **Structural demand** — does this problem exist over the long term, or only because a hot project appeared?
2. **Verifiability** — can it be validated through a demo, flowchart, transaction records, logs, user interviews, or a reference implementation?
3. **Minimal entry point** — can you produce a problem breakdown, flowchart, mock, repo skeleton, or minimal prototype within one week?
4. **Risk boundaries** — does it involve private keys, signatures, funds, identity, sensitive data, governance power, or irreversible actions?
5. **Follow-through** — can it naturally feed into a proposal, Hackathon challenge, or long-term handbook/research backlog?

## How to Use

1. Choose 2 directions of interest — write one sentence each: who is the real user?
2. Write one sentence: why is this hard without AI, and what is missing without Web3?
3. Judge whether it is closer to: product demo, developer tooling, protocol/standard, risk model, or research memo
4. Choose ONE main direction for deeper exploration; put others in backlog
