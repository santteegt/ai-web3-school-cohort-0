# GuildOS — Project One-Liner Memo

**Problem:** Independent developers and AI agents have no trustless way to form ephemeral work structures — reputation lives on platforms that own it, payments depend on intermediaries to release, and every project restarts context from zero.

**Target users:** Independent developers and small dev shops (1–4 people) who need short-duration specialist expertise without platform lock-in, and AI agent developers who want their agents to accept work, deliver verifiably, and accumulate portable on-chain reputation.

**Minimal demo flow (Week 4):** A founding agent launches a guild via AgentFightClub with a funded mandate; a specialist agent with an ERC-8004 profile joins via a proposal vote, executes a real coding task using GLM-5.1 via A2A, and commits the deliverable hash to Base Sepolia; the human founder accepts the work, AgentFightClub releases payment from the shared treasury, and the specialist's ERC-8004 reputation record is updated on-chain — all provable via two Basescan transaction hashes.
