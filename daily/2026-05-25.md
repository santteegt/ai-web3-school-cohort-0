### Evaluation chapter

- Evaluation is a very important part of an AIxWeb3 system because errors can affect assets, permissions, governance judgment, and on-chain execution
- If AI behavior is not measured repeatedly, it cannot be improved reliably.
- The goal of evaluation in AI for teams is to help them answer the question: did this change make key tasks more reliable? or Did it introduce new failure modes?
- Harness is a framework that repeatably runs evals over prompts, models, retrieval strategies, etc
- Even if an LLM can be used as a judge for evaluating open-ended answers, its outputs may have bias, miss issues, and can be influenced by output style
- It is important to maintain a Golden set that covers real tasks and key failure modes as well as regression tests that fixes historical problems in place and reruns them for every change.
- It is also important to add an observability module to an AI system that has the ability to observe system behavior online, so you know where real users fail, and what to add to the golden set
- Chapter notes live is `/knowledge-base/AIxWeb3/raw/Evaluation.md`

### Fine-tuning chapter

- It is useful for scenarios where a model needs to learn a certain format, style, domain task, or behavior pattern more consistently
- The goal is to make the model more consistent on a class of tasks, not to make it magically know the latest facts
- Goal, data, evaluation, must be properly defined before fine tuning
- A dataset is the core asset of fine-tuning. It needs a clear task definition, input sources, output standards, quality checks, and split strategy
- Fine tuning techniques include SFT and PEFT such as LoRA.
- In the AIxWeb3 space, fine-tuning can be used for specific tasks such as transaction-explanation style, governance-summary format, risk-label output, contract-comment style, and tool-call style
- But it cannot directly turn the model into a trusted execution layer
- Chapter notes live is `/knowledge-base/AIxWeb3/raw/Fine Tuning.md`

### Inference chapter

- Inference is in charge of delivering a usable answer under constraints
- It is a combined choice across latency, cost, context, stability, quality, privacy, operational complexity and deployment boundaries
- API Model vs Local Model + Quantization
- In AIxWeb3 systems, on-chain actions are hard to reverse, so **the inference service must leave auditable records**
- Chapter notes live is `/knowledge-base/AIxWeb3/raw/Fine Tuning.md`
