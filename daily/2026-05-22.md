## What I Read / Watched

<!-- Fill in after reading. Key ideas only. -->

- Read the `Context` chapter and highlighted the following key learnings:
  - Context is the entry point connecting models with real systems and their information
   - Context is the information space the model can see, use, and be influenced by in a current session
   - It is not simply long-text concatenation. It is an information governance problem. Each section should be properly labelled by source, freshness, permission, and trust level
   - A well-constructed Agent context include layers like Instructions, Tasks, Facts, Knowledge and Memory
   - For each source, it should maintain things like origin, update time, version and deprecation status
   - Context chapter notes live in the repo file `/knowledge-base/AIxWeb3/raw/Context.md`
- Read the `RAG` chapter and highlighted the following key learnings:
  - RAG is an evidence chain that aims to feed the model with external knowledge while making answers sourced, versioned, and bounded to prevent hallucinations.
  - RAG sits between the system/user Knowledge Base and the model. It helpts fetch relevant information and enrich the model context but it is not responsible for final execution
  - It is important to know that retrieved results are not fact but candidate evidence that helps the model respond to the user intent
  - Therefore a good RAG retriever component cannot look rely on pure vector search, but also other information/metadata in the session/context when fetching external knowledge.
  - Citations are a really important section of the RAG system as it connects model conclusions to sources, while also highlighting metadata as versioning, update time, provenance, etc
  - RAG chapter notes live in the repo file `/knowledge-base/AIxWeb3/raw/RAG.md`
