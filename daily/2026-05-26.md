### Watched the Long-term Memory for AI Agents - session replay

- Things learned:
  - Memory builds through work, and forms continuity for the user, meaning the user doesn't have to form context over and over across sessions
  - Memory is useful to store user preferences and useful past context, while the user has control and visibility boundaries over it
  - Memory management requires strong RAG & Context engineering techniques
  - Common memory workflow: Preserve useful state (transform into vectors using an embedding model) / Localte & Return (vector search) / Retrieve then Generate (re-rank?)
  - Memory lifecycle: Capture selectively / Retrieve selectively / Revise continuously
  - Memory should also be benchmarked in terms of recall
  - Claude code CLAUDE.md is an example of a project's evolving long-term memory
  - Agent Memory has two axes: 
    - Time: short-term memory (context) or long-term memory
    - Content: episodic, semantic, procedural
    - Orthogonal: scratchpads running on the side

- Questions to ask
  - Are third-party memory plugins implementing a graph-based knowledge memory store better than pure plain text memory engines that come by default on agents like Openclaw?
  - What are good benchmarking frameworks to test your own memory layer?
