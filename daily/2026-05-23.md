#### Open Agentic Economy

- AI is a new kind of economic participant, while Ethereum acts as the coordination layer to verify commitments/value creation/transfer
- Agents can't use the same infrastructure as humans for coodination. A new kind of infra will shape how the agent economy works, and it will compound as AI scales
- Ethereum provides property rights and trustless coordination for both humans and agents
- Ethereum is human-centric (e.g. human verify rules and agreements) and a neutral layer where agents can coordinate
- ERC-8004 helps to establish agent trust through identity, reputation and validation
- CROPS as design principle (Censorship-resistant, OSS, Privacy, Security)
  - it enabled things like identity, reputation, payments, enforceable agreements.

### Handbook Reading

- Today, I just read the Agents chapter. It provided the following key learnings:
  - An Agent is an AI system that can continuously call tools, read state, and adjust steps around a goal.
  - An Agent is not autonomy itself; it is a constrained execution loop
  - A usable Agent must know:
	 - what it can do / what it cannot do
	 - how to verify completion
	 - how to stop on failure
	 - who can audit what it has done
  - Agents in the Web3 scenery sits between model capability and on-chain execution
