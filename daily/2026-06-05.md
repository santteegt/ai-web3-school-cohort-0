## What I Read / Watched

- Agent Planning aims to break down a goal into steps: identifying which tools, read/write classification, confirmation needs (high-risk policy-checking boundaries matter in Web3 workflows), and completion verification.
- A2A protocol provides a communication and collaboration layer that handles agent discovery, negotiation, and task exchange — complements agent identity.
- **Five-layer Agent Context**: a structured framework for organizing agent information by source, trust level, and refresh rate — from instruction rules through system memory.
  - **Instruction** layer: system rules
  - **Task** layer
  - **Fact** layer: currently verifiable state from external systems
  - **Knowledge** layer: historical reference data
  - **Memory** layer: user preferences and configuration
