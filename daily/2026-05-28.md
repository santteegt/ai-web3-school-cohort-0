- Starting the day, I did a quiz on Agent planning. It helped me draw a clear mental model of the architecture of an agent acting in the Web3 space. The 8-step reference pattern is:
The 8-step pattern:
   1. **User provides goal and constraints** — scoped, not vague
   2. **Agent reads context and generates a plan** — candidate route, not authorization
   3. **System splits plan into read-only steps and candidate write steps** — the critical safety gate
   4. **Read-only tools run automatically** — RPC queries, balance checks, price lookups
   5. **Write tools enter policy checks** — permission validation, spending limit enforcement
   6. **Simulation shows on-chain impact** — before any signing, model the transaction outcome
   7. **User confirms high-risk actions** — irreversible actions require explicit consent
   8. **Wallet/Smart Account executes; logs record each step and final state** — full auditability

- Watched the replay of Cobo Agentic Wallet:
   - Giving agents wallet permissions introduces the following risk scenarios:
      - Prompt injection
      - Unscoped authority - broad wallet access
      - Shadow operations - wallet actions outside user visibility
      - Zombie permissions - stale never-expiring keys become persistent attack surfaces 
   - How control is maintained?
      - Give agents scoped authorization
      - Bind execution to it
   - Don't give agents wallets - **Give them Pacts**
   - Three components that redefines how agents go on-chain:
     1. **Multi-party Computation**: secures the private key by splitting PK between agent, user and COBO
     2. **Pact Authorization Protocol**: structured delegation with intent policy and auto-expiring
       - Intent
       - Execution plan: transparent reviewable roadmap
       - Policies: budget, approvals, etc
       - Completion conditions
     3. **Recipe Skill Layer**: verified execution paths for real on-chain tasks
       - Offers the first recipe-driven skill layer for AI agents
       - skill "capsules" embedding domain knowledge for specific onchain tasks
       - agents execute along predefined, validated workflows
