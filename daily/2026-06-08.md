## What I read / watched

- When using *Multimodal models*, visual interpretation alone is insufficient for authoritative decisions in Web3 workflows. This *distinction between capability and trust is essential for building safe AI agents in blockchain contexts*.
- The **verification chain** is a multi-layer system design pattern where each step in the pipeline adds a check before an action is taken.
  - Distribute security responsibility across *six layers* instead of relying on prompts alone
    - Prompt → Context → Model → Code → Guard/Simulation → Human check
- ERC-8004 is a composable registry foundation for agent identity and reputation. It doesn't guarantee capabilities or centralize trust — instead, applications build their own filtering and ranking rules on top. Perfect for GuildOS capability registration!
- In AI x Web3 scenarios, LLM outputs should be converted into verifiable objects (schema, parameters, citations) before execution
  - User goal → LLM reasoning → Verification/conversion to verifiable objects → On-chain execution
