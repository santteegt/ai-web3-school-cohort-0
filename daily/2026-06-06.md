- a key principle for practical **Proof of Inference** design, is that the **strongest proof is not always best**. The right method depends on the specific goal, not just model size:
  - Signed logs may suffice for service accountability
  - TEE for binary+model verification
  - ZK for minimal third-party trust
- The set of rules defining which agent or user can call which tools, under what authorization scope, and with what confirmation requirements — the most underestimated issue when MCP enters real products.
- Reviewed Vefifiable Compute concepts via quiz:
  - Definition — Understanding Verifiable Compute as off-chain execution with on-chain verification
  - Example — Recognizing when to apply it (expensive AI inference needing trust)
  - Boundary — Knowing what it doesn't require (public on-chain posting of raw data)
  - Process — The correct flow: execute off-chain → prove → verify on-chain
  - Application — Applying it to real scenarios like AI risk scoring in lending

**What I did today:**

Completed the x402 × Cobo Agentic Wallet payment loop prototype (`experiments/caw-payment-loop/`). The full loop runs end-to-end on Base Sepolia: agent gets a 402 challenge from the stub server, decodes the `PAYMENT-REQUIRED` header, signs an EIP-712 `TransferWithAuthorization` (EIP-3009), retries with the `PAYMENT-SIGNATURE` header, and receives a 200 with an audit log entry and on-chain tx hash via the x402.org facilitator. CAW Pact constraints are enforced (per-call ceiling $0.01, rolling 24h budget $0.10, 1-hour auto-revoke). Risk boundaries are documented. Currently running with a local signer workaround because `X402_INSUFFICIENT_BALANCE` turned out to be a Cobo node indexing outage on Base Sepolia — not a balance API bug. Restore to the production CAW path (`txApi.payment()`) is documented and ready to flip once Cobo confirms infrastructure recovery. Also reviewed Verifiable Compute and MCP permission model concepts, and catalogued all 19 WCB tasks due tomorrow into today's daily note.

**What I learned:**

- The `X402_INSUFFICIENT_BALANCE / available: "0"` Cobo error is a node indexing issue, not a balance bug. Workaround: bypass the CAW signing path with a local viem signer — the rest of the x402 flow is identical.
- Pact-scoped API keys are a clean permission boundary: once a human approves a pact, the agent uses only the pact key (scoped to destination address, amount ceiling, and time window) — never the owner key. Maps directly to GuildOS's controllable-agent requirement for the Cobo track.
- For Proof of Inference, the right proof mechanism depends on the goal, not model size: signed logs for service accountability, TEE for binary+model verification, ZK for minimal third-party trust.
